1. Basics

Print

  • Once you've opened a python interpreter you can type any valid python code and hit enter. The canonical starting point is the "Hello World". Paste the following into your interpreter (after the >>>) 
print "Hello World"

Python syntax is designed to be very readable. We have a function, print, followed by the argument to the command, telling it what to print. In this case, we passed the string "Hello World", but you can pass any type of object to the print command, and Python will spit out a string representation of that object.

Since "Hello World" is already a string, print just spits it back at us. But lets pass some other types of objects to the print command. Other object types include:

type python name
integer int
floating decimal point float
booleans bool


Let's print some of them!

# lets print an integer
print 2

# an alternative way call to print (this looks like a more traditional function call)
print(2)

# lets print a float
print 2.1

# how about a boolean?
print True

How does python know how to print all these different types of objects?

  • Every object in python has string representation of itself. When you pass any object to the print command, it converts it to that string, and displays it to you. This might seem simple, but its a fundamental concept. print takes any kind of object and returns it's string interpretation.

How can we know the type of an object?

  • Just like with the print statement, python has a very intuitive way of doing this. It's called the type function.

Type

# how do we know that 2.1 is really a float?
type(2.1)

# type can be called on any type!
type(True)
type("Hello World")

In the same way that every python object has a string representation, every python object must also have a type. Just as the print function returns the string interpretation, type returns the object type. It's a very useful command!
